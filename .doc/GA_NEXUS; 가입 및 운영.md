#  지점 통합 관리 시스템 가입/운영 로직 (Manager & Agent)

본 가이드는 지점장(Admin)의 시스템 구축부터 에이전트(Agent)의 합류 및 승인 프로세스를 단계별로 설명합니다.

---

## 1. 전체 프로세스 흐름 (Workflow)

1. **지점장(Admin) 등록**: 구글 계정으로 1초 만에 로그인 및 본인만의 관리 시스템 생성.
    
2. **초대 코드 생성**: 지점장 대시보드에서 우리 지점 전용 **'초대 코드'** 발급 및 공유.
    
3. **에이전트(Agent) 신청**: 초대 코드를 받은 에이전트가 가입 신청서 작성.
    
4. **지점장 승인**: 신청 내역 확인 후 지점장이 [승인] 버튼 클릭.
    
5. **협업 시작**: 승인된 에이전트에게 캘린더 대시보드 권한 즉시 부여.
    

---

## 2. 지점장(Admin) : 시스템 마스터 권한 확보

지점장님은 복잡한 가입 절차 없이 기존에 사용하시던 **구글 계정**을 그대로 사용합니다.

- **가입 방식**: Google OAuth 2.0 (구글로 시작하기).
    
- **시스템 소유**: 로그인 직후 '우리 지점 명칭'을 입력하면, 해당 지점의 고유 ID와 연결된 전용 캘린더 DB가 자동 생성됩니다.
    
- **초대 코드 관리**:
    
    - 랜덤 또는 지정 코드를 생성합니다 (예: `MERITZ_SEOUL_01`).
        
    - 해당 코드를 카카오톡 단톡방에 공유하여 인원을 모집합니다.
        

---

## 3. 에이전트(Agent) : 가입 신청서 양식 구성

에이전트가 초대 코드를 입력했을 때 나타나는 **입력 폼(Form)** 구성안입니다. 이 정보들은 지점장님의 승인 대기 리스트에 데이터로 쌓이게 됩니다.

### 📋 신규 에이전트 가입 신청서

|**항목**|**입력 타입**|**비고**|
|---|---|---|
|**초대 코드**|Text|지점장에게 전달받은 코드 입력 (유효성 검증)|
|**성함**|Text|실명 입력|
|**소속 지점**|Text|본인의 지점명 확인 (자동 입력 또는 선택)|
|**생년월일**|Number (6자리)|예: 950101 (신원 확인용 코드)|
|**휴대폰 번호**|Tel|하이픈(-) 제외 숫자만 입력|
|**개인정보 활용 동의**|Checkbox|[필수] 지점 일정 관리 및 신원 확인 목적 동의|

> **💡 UX 포인트**: 신청 버튼 클릭 시 "지점장님의 승인을 기다리고 있습니다. 승인 후 카카오톡으로 알림이 전송됩니다."라는 안내 문구를 노출하여 신뢰감을 줍니다.

---

## 4. Supabase 백엔드 로직 (Invisible Logic)

데이터베이스에서는 다음과 같은 작업이 자동으로 처리됩니다.

- **Step 1. 유저 생성**: 에이전트가 신청서를 제출하면 Supabase Auth에 계정이 생성되지만, `is_approved` 상태값은 `false`로 저장됩니다.
    
- **Step 2. 관리자 알림**: 지점장(Admin)의 대시보드에 **'신규 승인 대기 1건'** 배지가 생성됩니다.
    
- **Step 3. 권한 업데이트**: 지점장이 승인하면 `is_approved`가 `true`로 바뀌며, 이때부터 보안 정책(RLS)에 의해 막혀있던 캘린더 데이터가 에이전트 화면에 보이게 됩니다.

## 5. 추천 추가 기능: 지점장 대시보드(Admin Panel)

승인 이후 지점장님이 멤버들을 효율적으로 관리할 수 있는 기능입니다.

- **멤버 리스트 관리**: 현재 승인된 인원의 목록과 연락처를 한눈에 확인.
    
- **권한 회수 (퇴사 처리)**: 퇴사자가 발생할 경우 '권한 정지' 버튼 클릭 시 즉시 캘린더 접근 차단.
    
- **코드 재생성**: 보안을 위해 초대 코드를 주기적으로 변경하거나, 1회용 코드로 발급하는 기능.[[GA_NEXUS; UI-UX]][[GA_NEXUS ; PROCESS]]
